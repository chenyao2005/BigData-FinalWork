# nyc-longform-github 目录结构说明（队友交接用）

本仓库为**可独立部署的静态长文主站**：根目录即站点根路径，含叙事首页与 `apps/` 下各章嵌入子应用。部署到 **GitHub Pages** 时，整仓内容作为站点根目录发布（若仓库名为 `nyc-cityexplore`，线上 URL 形如 `https://<用户名>.github.io/nyc-cityexplore/`）。

---

## 一、顶层目录树（逻辑视图）

```
nyc-longform-github/
├── .gitignore              # Git 忽略规则（系统垃圾文件、日志等）
├── .nojekyll               # 禁用 GitHub Pages 默认 Jekyll 构建（纯静态站点必需）
├── README.md               # 仓库说明：预览方式、部署、目录导读
├── index.html              # 主站入口 HTML
├── main.js                 # 主站逻辑：读文案、渲染章节、嵌入 iframe
├── styles.css              # 主站全局样式与章节排版
├── _restructure-apps.ps1   # 一次性迁移脚本（历史用途）：把旧散落目录收拢到 apps/
├── data/                   # 主站配置与文案（JSON）
├── output/                 # 第四篇章「环境」依赖的全局汇总数据（仓库根目录级别）
└── apps/                   # 按叙事顺序编号的嵌入可视化（每章一个子文件夹）
    ├── 01-population-yearly/
    ├── 02-population-rings/
    ├── 03-education-level/
    ├── 04-safety-linebox/
    ├── 05-economy/
    ├── 06-transport/
    └── 07-environment/
```

以下「文件」若下称「目录」，表示该路径为文件夹及其用途说明。

---

## 二、根目录文件

| 文件 | 功能说明 |
|------|----------|
| **index.html** | 主页面骨架：挂载 `#hero`、`#main-content`、引用 `styles.css` 与 `main.js`（带 `?v=` 便于缓存刷新）。 |
| **main.js** | 拉取 `data/site-content.json`，渲染 Hero、四章内容及终章；按章节拼接 iframe、交互指南、段落；第四篇章 iframe 带缩放样式常量。 |
| **styles.css** | 主站视觉：字体、章节卡片、`.iframe-wrap`、第四篇章缩放容器等；**不改子应用内部样式**。 |
| **README.md** | 给人看的说明：本地预览命令、GitHub Pages、`apps/` 与各篇章对应关系、如何改文案。 |
| **.gitignore** | 忽略 `.DS_Store`、`Thumbs.db`、`*.log` 等，避免无效文件进仓库。 |
| **.nojekyll** | 空文件即可；告知 GitHub Pages **不要跑 Jekyll**，否则以下划线开头的文件名等可能被错误处理。 |
| **_restructure-apps.ps1** | **仅需在本仓库根目录、历史迁移时执行一次**：将过去的 `nyc-viz-summary/…`、`nyc-pulse` 等移动到 `apps/01-…07-…`。若目录已迁完，可忽略；勿重复执行以免报错。 |

## 三、`data/` — 主站配置与文案

| 文件 | 功能说明 |
|------|----------|
| **site-content.json** | **核心配置**：首页标题与引言、四章标题/导语/交互指南、各 iframe 的 `src`/`height`、章节内段落与图表解说文案；改叙事或 iframe 地址**优先改此文件**。 |
| **integration-manifest.json** | **模块索引**：记录各篇章对应 `apps/` 入口 HTML、代表性数据路径，便于打包或交接时快速定位。 |
| **profile-chart-placeholder.json** | 终章画像图表相关占位或默认数据（若主站 `main.js` 有引用则生效）。 |

---

## 四、`output/` — 第四篇章全局数据

环境子应用通过 **`apps/07-environment/main.js`** 内 **`../../output/…`** 读取此处文件（相对「环境应用目录」向上两级到仓库根）。

| 文件 | 功能说明 |
|------|----------|
| **final_data.environment.json** | 按邮编聚合的环境指标等主数据。 |
| **score4.csv** | 综合打分等表格数据（与环境雷达/排序相关）。 |
| **facilities_by_zip.csv** | 公共设施等按邮编汇总。 |
| **environment_by_zip.csv** | 环境维度按邮编的补充表（若应用内有引用）。 |

更新环境篇上游流水线时，替换这些文件即可（注意体积与编码）。

## 五、`apps/` — 嵌入子应用（按章节编号）

每个子目录为**独立静态前端**，通常包含 **`index.html` + 打包后的 `assets/` + 本地 `data/` 或大 JSON**。主站在 `site-content.json` 里用 **相对路径**（如 `./apps/03-education-level/index.html`）嵌入。

| 目录 | 篇章 | 简要说明 |
|------|------|----------|
| **01-population-yearly** | 第一篇章 | 人口年度变化等时序图；含 `main.js`、`style.css`、多处 `data/`（原始/处理后的 CSV、JSON）。 |
| **02-population-rings** | 第一篇章 | 人口圈层相关可视化（Vite 打包：`assets/*.js`、`assets/*.css`）；`data/` 含 GeoJSON、统计 JSON 等。 |
| **03-education-level** | 第一篇章 | 教育水平三联图（打包资源在 `assets/`）；`data/` 为学校与边界等地理/表格数据。 |
| **04-safety-linebox** | 第一篇章 | 公共安全 borough 箱线图等（打包在 `assets/`）；`data/` 含犯罪与地理边界数据。 |
| **05-economy** | 第二篇章 | 经济账本（Vue 应用）；`assets/` 为打包 JS/CSS；**`nyc_master_data.json`** 为体积较大的主数据集；含 `index.html` 内针对 GitHub Pages 的 `<base>` / 路径修正脚本；`icons.svg`、`favicon.svg` 等静态资源。 |
| **06-transport** | 第三篇章 | 时空脉搏（地铁/矩阵等）；`assets/` 为打包文件；根目录另有若干 **矩阵 JSON、GeoJSON、CSV** 供运行时读取。 |
| **07-environment** | 第四篇章 | 微观生态地图：`main.js`、`styles.css`、`index.html`；`data/nyc_zip_boundaries.geojson`；**汇总 JSON 读仓库根 `output/`**；`submission.html`、`final_data.sample.json` 等为课程/演示备用页。 |

**维护提示**：子应用若重新打包，一般会替换 `assets/` 下带哈希的文件名，记得同步更新对应 **`index.html` 里的 script/link 引用**，并酌情递增主站或子页上的 **`?v=`** 参数以免浏览器缓存旧文件。

## 六、本地预览与部署要点

1. **必须在 HTTP 服务下打开**（如 `python -m http.server`），不要用 `file://`，否则 `fetch` JSON 会失败。  
2. **GitHub Pages**：仓库 Settings → Pages → 从 **`main`** 分支 **`/(root)`** 发布；若页面异常，确认存在 **`.nojekyll`**。  
3. **修改叙事或 iframe 列表**：编辑 **`data/site-content.json`** 后刷新即可（必要时递增 **`index.html`** 里 **`main.js` / `styles.css` 的 `?v=`**）。
